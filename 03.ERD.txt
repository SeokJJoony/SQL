*키와 제약조건: 개체 속성 표기법
테이블은 데이터를 저장하는 서랍과 같은 역할을 함.
일련의 규칙인 키와 제약 조건을 가짐

기본키: 데이터를 식별하는 '식별자' 역할을 함. 중복 값을 가질 수 없고 NULL 값을 가질 수 없음. 
         변경될 수 없음. (= 주민등록번호)

고유키: 행에서 유일한 값을 갖는 데이터 값으로 구성된 열

외래키: 다른 테이블과 연결을 위한 열

복합키: 기본키이자 외래키를 의미

제약조건이란? 테이블을 생성할 때 명시하는 조건 ->데이터를 인가되지 않은 방법으로 변경할 수 없도록 보호
 (무결성 준수)

제약조건 종류
기본키: UNIQUE +NOT NULL을 만족하여 테이블을 대표하여 각 행을 유일하게 식별하는 값
외래키: 열 값이 부모 테이블을 참조 열 값을 반드시 참조/ 참조되는 열은 유니크하거나 기본 키임(null 허용가능)
고유키: 중복된 값을 허용하지 않음, 유일한 값으로 존재(null 허용 가능)
NOT NULL: null 값을 허용하지 않음, 값 입력 필수
CHECK: 범위나 조건을 설정하여 지정된 값만 허용

*관계: 개체 관계 표기법
1 : 1대1
0 또는 1 : 0일수도 잇고 1대1일수도있음
1 이상 : 1이거나 여러개와 관계를 갖음
0 이상 : 0이거나 여러개와 관계를 갖음

실선: 필수관계
점선: 선택적 관계
